# Transcend repositioning experience concept

This is a lightweight GitHub Pages website for a PMM interview assignment.

## Narrative

The concept reframes Transcend's solution story from primarily company-type navigation to a buyer outcome and maturity narrative:

1. Automate privacy rights and consent
2. Govern customer data at the systems layer
3. Activate AI responsibly

Company type navigation, such as AI, B2B SaaS, fintech, media, consumer, mid-market, and enterprise, remains useful as a secondary path, but the main story becomes the buyer journey from privacy operations to AI-ready compliant data activation.

## Included

- Interactive before-and-after maturity experience
- Three stages of customer maturity
- Embedded Transcend Pulse audio cue
- Restrained visual system inspired by Transcend's public site feel
- Clear disclaimer that this is not official Transcend brand guidance

## Files

- index.html

## GitHub Pages instructions

1. Create a new GitHub repository.
2. Upload all files from this folder.
3. Go to Settings.
4. Go to Pages.
5. Select deploy from branch.
6. Choose main and root.
7. Save.
8. Use the public GitHub Pages URL in your assignment.

## Suggested talk track

"I kept company-type navigation as a useful secondary path, but repositioned the primary story around buyer maturity. The customer first needs to automate privacy rights and consent, then govern customer data at the systems layer, and finally activate AI responsibly on a compliant data foundation. The Pulse audio cue is intentionally restrained. It only appears once the workflow is mature enough to signal verified completion."
